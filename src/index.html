What is Github

git•hub - /'ɡɪtˌhʌb/

GitHub is the best way to collaborate with others. Fork, send pull requests and manage all your public and private git repositories.

The Foundation

  • Github created a social network around technology that was created by Linus Torvalds called Git.
  • Git is source control management software.
  • It's written in Ruby, CoffeeScript, & Erlang.

The Purpose

  • Collaboration
  • Revisioning
  • Learning
  • Code Hosting
  • Code Review

Github Workflow

Someone has an error in their code.

Step 1. Create an Issue

  • A useful title.
  • A description.
  • If you are legit, some ideas around a solution.
  • You can also assign, label, and milestone it if you have rights.

Step 2. Discuss It

Typically some healthy discussion around an issue occurs.

Step 3. Pull Request

Someone finally fixes the error in their fork, and requests the fix be merged in.

Forks typically have a message referencing the error.

Step 4. Code Review

People review the code, comment on line numbers, API's and more.

Step 5. Pull Request Merge

A repository team member merges the pull request into the main repository.

Step 6. High Fives

Everyone gets the fix. The software is improved. Food tastes better. Colors are brighter.

Why It's Better

  • Community descisions, remove bureaucracy
  • APIs are typically better and more thorough.
  • Issues and documentation are right there with the code.

Revisioning

Git gives us the revisioning power we need to rollback or see the history of a project.

Github gives us a sexy interface for it.

In Context Review

Comments can be made right on a line of code.

Or on an issue or pull request.
